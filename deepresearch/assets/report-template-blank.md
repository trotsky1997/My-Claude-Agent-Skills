# QA Checklist（交付前必跑）

## A. 范围与结构
- [ ] 主问题与时间窗写清楚，Non-goals 明确
- [ ] KIQ ≤ 7 且每个都被回答
- [ ] 结论先行（Key Judgments 在前）

## B. 证据与引用
- [ ] 每条关键判断都有证据锚点（不是堆链接）
- [ ] 关键数字/事实均可追溯到原始来源或推导链
- [ ] 引用信息足够定位（标题/机构/日期/URL/访问日期/页码或段落）
- [ ] 关键网页已存档/截图/下载，防链接失效

## C. 推理与替代解释
- [ ] 明确区分事实 vs 判断 vs 推断
- [ ] 至少 1 个替代解释被认真呈现并评估
- [ ] 关键假设显式列出，脆弱性与验证路径明确
- [ ] 冲突证据已记录并解释取舍

## D. 不确定性表达
- [ ] 每条关键判断写了 Likelihood + Confidence
- [ ] 概率词使用一致；避免含混卸责词（如 apparently/reportedly 无说明）
- [ ] 写清"什么证据会改变结论"（翻转条件）

## E. 可用性与行动性
- [ ] 给出可执行建议或选项（含风险/代价）
- [ ] 有监测指标/信号（signposts），方便持续更新
- [ ] 报告长度与细节符合受众（管理层可 5 分钟读懂；附录可审计）
